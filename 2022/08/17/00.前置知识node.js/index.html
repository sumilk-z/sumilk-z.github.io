一些基础问题

为什么JavaScript可以在浏览器中被执行？

答：因为浏览器中内置了JS解析引擎，例如chrome浏览器的V8引擎，IE浏览器的chakra，Safri浏览器的JSCore等。

为什么JavaScript可以操作DOM和BOM？

答：因为每个浏览器都内置了DOM，BOM这样的API函数，JavaScript可以调用他们。包括DOM API，BOM API，Ajax API。

什么是JavaScript运行环境？

运行环境包括：JavaScript解析引擎，以及可调用的内置API。 浏览器是典型的JavaScript运行环境；

什么是node.js?

Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine.

==Node.js是一个基于Chrome V8引擎的JavaScript运行环境。==

  • 浏览器是JavaScript的前端运行环境

  • node.js是JavaScript的后端运行环境

  • 在node.js中无法调用DOM，BOM，Ajax的API。

node.js可以做什么？

node.js作为一个JavaScript的运行环境，仅仅提供了基础的功能和API。但是基于node.js提供的基础功能，可以构建很多强大的工具和框架。

node.js学习路径

浏览器中的JavaScript学习路径：

JavaScript基础语法+浏览器内置API（DOM，BOM）+第三方库（jQuery,art-template等）

Node.js的学习路径

JavaScript基础语法+Node.js内置API模块（fs,path,http等）+第三方API模块（express，MySQL等）

node.js中的模块化规范

CommonJS模块化规范，规定了：

  • 每个模块内部，module变量代表当前模块
  • module变量是一个对象，他的exports属性是对外接口
  • 加载某个模块，其实是加载该模块的module.exports属性。require()方法用于加载模块。

npm与包

包：即第三方模块。基于内置模块封装出来的。提供了更高级，更方便的API

国外有一家IT公司，叫做npm.Inc，这家公司旗下有一个非常著名的网站：https://www.npmjs.com,它是全球最大的包共享平台。

这个公司提供了一个包管理工具，可以使用这个包管理工具，从https://registry.npmjs.org/服务器上把需要的包下载到本地使用。

这个包的名字叫做 ：Node Package Manager，简称npm包管理工具，这个工具随着node.js的安装一起被安装到了用户的电脑上。

初次安装包完成后，在项目文件夹下多了一个叫做node_modules的文件夹和package-lock.json的配置文件。

其中：

  • node_modules 文件夹用户存放所有已安装到项目中的包，require()导入第三方包时，就是从这个目录中查找并加载包。
  • package-lock.json配置文件用来记录node_modules目录下的每一个包的下载信息，例如包的名字，版本号，下载地址等。

不要手动修改这node_modules或者package-lock.json文件中的任何代码，npm包管理工具会自动维护他们。

包管理配置文件package.json

npm规定，在项目根目录中，必须提供一个叫做package.json的包管理配置文件，用来记录与项目有关的一些配置信息，例如：

  • 项目的名称、版本号、描述等
  • 项目中都用到了哪些包
  • 哪些包只在开发器件会用到
  • 哪些包在开发和部署的时候都需要用到

在团队合作开发中，这个很重要，不用上传第三方包。而只需要告知使用了哪些第三方包即可。

npm包管理工具提供了一个快捷命令，可以创建package.json文件

  • 只能在英文的目录下成功运行，所以项目文件夹的名称一定要使用英文，不能使用中文，出现空格。
  • 运行npm install命令安装包的时候，npm包管理工具会自动把包的名称和版本号，记录到package.json中。

包的分类

项目包：包括开发依赖包（devDependencies)和核心依赖包（dependencies）

全局包，安装目录位于：C:\Users\用户名\AppData\Roaming\npm\node_modules，安装的时候需要指定-g

包的规范

一个规范的包，组成结构必须符合三点：

  • 包必须以单独的目录而存在
  • 包的顶级目录下必须要包含package.json这个包管理配置文件
  • package.json中必须包含name,version,main这三个属性，分别代表包的名字，版本号，包的入口
npm发布包
  • 需要在npm网站上注册一个账号
  • 然后在终端中登录该账号npm login
  • 记得将服务器切换为官方的服务器，而不能是镜像服务器
  • 切换到包的根目录下，执行命令 npm publish就可以发布包到npm上
  • 执行命令npm unpublish 包的名称 --force就可以将发布的包删除（只能删除发布72小时以内的包，且24小时以内不能重复发布包）
模块的加载机制

  • 优先从缓存中加载

    ​ 模块在第一次加载后会被缓存，多次调用require()不会导致模块的代码被多次执行。

  • 自定义模块的加载机制

    使用require()加载自定义模块时，必须指定以./或者../开头的路径标识符，如果没有，node会把它当作内置模块或者第三方模块进行加载。

    如果省略了文件的扩展名，则node.js会按顺序依次尝试加载

    • 按照确切的文件名进行加载
    • 补全.js扩展名进行加载
    • 补全.json扩展名进行加载
    • 补全.node扩展名进行加载

  • 加载包的时候，先找到package.json中的main属性，加载main属性所指向的文件，否则就加载index.js文件，如果连index.js文件都没有则报错。
